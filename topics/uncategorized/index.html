windows上的mysql安全设置

Posted on the 八月 14th, 2015 under 默认 by admin

前言

本教程中的安全设置无法保证你百分百就不被黑。
经过本教程的设置 你的mysql安全会提高一个层次
尤其是在多用户的情况下 特别是对windows上面的虚拟主机管理系统很有效果
比如 华众 星外 kangle 慧琳

正题开始

在windows上面 mysql服务默认使用的是系统账号SYSTEM
所以我们先从这个入手 给mysql重新指定一个普通用户去登录
本文当中使用的环境是windows2003 高版本windows 比如2008什么的都同样适用

设置步骤

  1. 给系统添加一个普通用户
  2. mysql目录权限设置
  3. 设置mysql使用这个普通用户登录

详细图文

首先我们看下面的图 可以看出当前mysql使用SYSTEM用户进行登录的

一旦你mysql的root密码被破解 黑客就可以轻松的使用SYSTEM用户权限
那么我们要做的就是 让mysql以普通用户去登录
就算你服务器其他地方的安全设置不合理 导致mysql的root密码被破解
就算黑客拿到这个密码 也不能进一步的对服务器造成威胁

给系统添加一个普通用户

这个超级简单，但还是要详细的截图出来 会了的可以略过
我的电脑-右键-管理

本地用户和组-用户-空白处右键-选择新用户

在新用户窗口 填写这个新用户的用户名和密码
我这里要创建的用户名是mysqluser 这个根据你自己喜欢来创建就行了没要求
设置的密码记住 一会要用到 一定要选择密码永不过期 如下图 填好了点创建就行了

现在我们查看下刚刚创建的用户

然后选择这个用户-右键-属性

找到 隶属于-Users-删除-应用
意思就是 让我们新建的这个用户不属于任何组 如下图

mysql目录权限设置

首先设置mysql主程序目录权限 也就是程序安装目录
比如我的安装目录是 D:\Program Files\MySQL\MySQL Server 5.5
那么就设置这个目录的权限 仅保留administrator 和SYSTEM 剩下的全部删除
然后添加刚刚添加好的独立用户给予 读取和运行权限 列出文件夹目录 读取 这三个权限 如下图

现在开始设置mysql数据库文件目录 就是你的数据库存储的目录
可以通过查看my.ini查询到
my.ini 在mysql的安装目录可以找到
打开my.ini 搜索datadir= 可以可以找到你的数据库文件存放目录了
如下图

现在我们找到了 数据库存放目录是

C:/Documents and Settings/All Users/Application Data/MySQL/MySQL Server 5.5/Data/

但为了以后的服务器目录安全设置 我们需要把这个目录转移走
比如E盘 D盘 随你个人喜好来做
我这里就根据个人喜好 把数据库存放目录修改为E:/MySQL Datafiles
如果你也想要修改 那么还需要把 

C:/Documents and Settings/All Users/Application Data/MySQL/MySQL Server 5.5/Data/这里面所有的文件拷贝到新目录，比如E:/MySQL Datafiles/

如下图


辅助数据库文件后 可以修改my.ini的路径了
我们改为
datadir=”E:/MySQL Datafiles/”

保存好my.ini后 可以去设置mysql数据库文件存放目录的权限了
保留administrator 和SYSTEM的权限 剩下的全删除 然后给mysqluser 完全控制的权限 如下图

设置mysql使用这个普通用户登录

开始-管理工具-服务

找到mysql-右键属性

选择登录-此账户-输入刚刚创建的账号和密码，比如我创建的是mysqluser 然后点应用



最后一步 服务-mysql-重启动

至此mysql已普通用户登录的设置就完成了
任务管理器查看这个进程 可以看到已经在使用普通用户在登录了
