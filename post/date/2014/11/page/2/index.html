linux安装SVN服务器+配置过程

Posted on the 十一月 9th, 2014 under 默认 by admin

测试环境

本人电脑自建 VMware CentOS 5
虚拟机配置 内存2G CPU2颗 北京20M光纤
因为linux发行版众多 可能有些命令并不通用

subversion安装

登录你的ssh 执行如下命令进行安装

yum install subversion


然后提示需要下载3.7M的数据 输入Y进行下载和安装

到这里就安装成功了

subversion配置

可以先查看下subversion的安装位置

rpm -ql subversion

创建svn版本库目录

mkdir -p /var/svn/svnrepos

创建版本库

svnadmin create /var/svn/svnrepos

创建完版本库后我们进入/var/svn/svnrepos目录 可以看到生成了如下图的几个文件
conf这个目录就是该svn版本库配置文件

conf目录介绍
首先进入conf目录
authz文件是权限控制文件
passwd是帐号密码文件
svnserve.conf SVN服务配置文件
如下图

配置账号密码

vi passwd

在[users]块中添加用户和密码，格式：帐号=密码，如下图 我配置的账号和密码 qiangren=

保存退出后配置权限

配置账号权限

vi authz

在末尾添加如下代码

[/]
qiangren=rw
w=r

意思是在版本库的根目录，qiangren用户对其有读写权限，如果你只想给读的权限那就设置为w

保存后退出在配置svnserve.conf

修改svnserve.conf文件

vi svnserve.conf

打开下面的几个注释
anon-access = read #匿名用户可读
auth-access = write #授权用户可写
password-db = passwd #使用哪个文件作为账号文件
authz-db = authz #使用哪个文件作为权限文件
找到realm = My First Repository修改为你的版本库所在目录
realm = /var/svn/svnrepos #版本库所在目录

保存退出 到此位置简单的配置完成了

启动svn版本库

svnserve -d -r /var/svn/svnrepos

到这里SVN服务器搭建完毕 可以正常使用了

总结

SVN服务器搭建就这些 下篇文章我会介绍如何在windows下面使用subversion客户端 以及遇到的问题
